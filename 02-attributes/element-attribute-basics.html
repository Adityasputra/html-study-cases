1. Basic Attribute Syntax

<element attribute="value">Content</element>

Components:

  • attribute name - lowercase, no spaces
  • = - equals sign (no spaces)
  • "value" - attribute value in quotes

3. Boolean Attributes (No Value Needed)

Some attributes don't need values - their presence means "true", absence means "false".

Examples:

Checked by default



<input type="checkbox" checked>
<input type="text" disabled>
<input type="text" readonly>
<input type="text" required>

4. Multiple Attributes on One Element

Elements can have multiple attributes separated by spaces.

<input
  type="email"
  id="user-email"
  name="email"
  placeholder="Enter your email"
  required
  maxlength="100"
>

5. Attribute Value Quotes

Double quotes (recommended):

<a href="page.html">Link</a>

Single quotes (also valid):

<a href='page.html'>Link</a>

No quotes (only for simple values with no spaces):

<input type=text>

When quotes are REQUIRED:

Values with spaces MUST be quoted:

✓ <img alt="A beautiful sunset">
✗ <img alt=A beautiful sunset>

8. Best Practices

  • ✓ Always use lowercase for attribute names
  • ✓ Always quote attribute values (prefer double quotes)
  • ✓ Use meaningful values (especially for id, class, alt)
  • ✓ Add alt attribute to all images for accessibility
  • ✓ Use semantic attribute names (name, not n)
  • ✗ Don't use spaces in attribute values without quotes
  • ✗ Don't duplicate attributes on the same element
  • ✗ Don't leave required attributes empty

9. Summary

Attribute Structure:

<tagname attribute="value">Content</tagname>

Key Points:

  • Attributes go in the opening tag
  • Format: name="value"
  • Multiple attributes are space-separated
  • Some attributes are boolean (no value needed)
  • Different elements have different valid attributes
  • Attributes modify element behavior and appearance
